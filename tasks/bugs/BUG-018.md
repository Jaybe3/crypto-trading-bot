# BUG-018: Dashboard shows Size=0, SL=0, TP=0 for open positions

**Severity:** HIGH
**Found:** 2026-02-06
**Found By:** User report + investigation
**Status:** FIXED

## Evidence

Dashboard JavaScript reads wrong field names:
```
236:                    <td class="text-right font-mono">${pos.size || 0}</td>
238:                    <td class="text-right font-mono text-gray-400">$${formatPrice(pos.stop_loss || 0)}</td>
239:                    <td class="text-right font-mono text-gray-400">$${formatPrice(pos.take_profit || 0)}</td>
```

API returns correct field names:
```json
{
  "size_usd": 40.0,
  "stop_loss_price": 0.265,
  "take_profit_price": 0.274
}
```

Fields don't match, so JavaScript displays 0/undefined.

## Root Cause

JavaScript template used incorrect field names (pos.size, pos.stop_loss, pos.take_profit)
instead of the actual API field names (pos.size_usd, pos.stop_loss_price, pos.take_profit_price).

## Fix

Three field name changes in dashboard/templates/index.html:
- Line 236: `pos.size` -> `pos.size_usd`
- Line 238: `pos.stop_loss` -> `pos.stop_loss_price`
- Line 239: `pos.take_profit` -> `pos.take_profit_price`

## Verification

```bash
# Confirm correct field names are used
grep -n "pos\.size_usd\|pos\.stop_loss_price\|pos\.take_profit_price" dashboard/templates/index.html

# Confirm no remaining wrong field names
grep -n "pos\.size\|pos\.stop_loss\|pos\.take_profit" dashboard/templates/index.html | grep -v "_usd\|_price"
```
