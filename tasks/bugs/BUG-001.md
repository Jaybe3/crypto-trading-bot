# BUG-001: Dashboard /api/knowledge/context returns HTTP 500

**Severity:** CRITICAL
**Found:** 2026-02-05
**Found By:** Audit Layer 3 (Integration) + Layer 4 (Runtime)
**Status:** FIXED

## Evidence
dashboard_v2.py:332 calls get_context_for_strategist() but KnowledgeBrain has get_knowledge_context() at line 467.
Runtime: curl /api/knowledge/context returns HTTP 500.

## Root Cause
Method name mismatch — dashboard calls a method that doesn't exist on KnowledgeBrain.

## Fix
dashboard_v2.py line 332: get_context_for_strategist() → get_knowledge_context()

## Verification
- grep confirms method name changed
- curl /api/knowledge/context returns HTTP 200 (if bot running)
- Full test suite passes
